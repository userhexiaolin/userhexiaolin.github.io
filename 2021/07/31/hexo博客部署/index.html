hexo博客部署

（注意：cmd界面要以管理员身份运行）

名词 是否必须 安装方式/释义
node.js 是 官网下载
git 是 官网下载
淘宝镜像源 否 npm install -g cnpm –registry=https://registry.npm.taobao.org
hexo 是 官网下载
mkdir 是 mkdir + 文件夹名，创建博客目录
hexo init 是 hexo init + 博客名称，生成一个新博客
图片路径转换插件 是 npm install https://github.com/CodeFalling/hexo-asset-image –save
hexo new 是 hexo new + 文章名，自动生成一个md格式的文件
（如果安装了路径转换插件，则会额外创建与文章同名的文件夹，
用来存储图片等文件）
hexo g 是 hexo generate；生成静态文件
hexo s 否 hexo server；本地调试
远端部署插件 是 npm install –save hexo-deployer-git
hexo d 是 hexo deploy；将博客推送到远端
markdown图片路径 是 文件——偏好设置——图像——插入图片时：1.复制到指定路径；2. ./$(filename); 3.对本地位置的图片应用上述规则；4.对网络位置的图片应用上述规则；5.优先使用相对路径

一、安装node.js

二、安装git

三、替换到国内源（下载速度慢时可选，替换后需将npm换成cnpm）

​ 在cmd界面输入：npm install -g cnpm –registry=https://registry.npm.taobao.org

四、安装hexo

​ 输入：npm install -g hexo-cli

五、创建一个空文件夹（必须是用命令创建文件夹，否则可能出错）

​ 在cmd界面输入：mkdir 文件夹名，cd到 刚创建的文件夹

六、创建博客

​ 输入：hexo init，自动生成博客。

七、安装图片路径转换插件

​ 打开博客主目录中的cofig.yml，找到post_asset_folder，将值设为“true“。
​ 输入：yarn https://github.com/CodeFalling/hexo-asset-image (yarn改成npm也可以。)

八、创建文章（安装图片路径插件后，会同时生成文章和同名文件夹）

​ 1、输入：hexo new + 文章名称，编辑文章内容。
​ 2、将图片放进md文件中后，右击图片，复制图片到”其同名文件夹“中，并将图片的绝对路径改为相对路径

​ 3、windows系统必须在路径前面加” ./ “，否则在博客中不会显示图片。

​ 4、改为绝对路径后，如果缩放图片，图片路径会发生改变，但是并没有影响。

​ 5、在.md文件中打开文件——偏好设置——图像——插入图片时：{

  • 复制到指 定路径；
  • ./$(filename)
  • 对本地位置的图片应用上述规则
  • 对网络位置的图片应用上述规则
  • 优先使用相对路径

九、生成静态文件

​ 输入：hexo g (即：hexo generate)

十、启动调试

​ 输入：hexo s （即：hexo server）

十一、连接github

​ （在git bash中输入：git config –global user.email “lin1009084929@gmail.com“ 和

​ git config –global user.name “userhexiaolin” ；即输入用户名密码，不然hexo不知道你的github账户，也就无法推送。）

十二、推送博客至github仓库

​ 安装git部署插件，输入：npm install –save hexo-deployer-git

​ 说明 ：部署前需要修改 _config.yml 配置文件，下面以 git 为例进行说明

​ 输入hexo d 或 hexo g -d ，将博客推送到远端。
